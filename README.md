# 🚀 AI-Ассистент диагност для ЧПУ Heidenhain TNC 620

Данный проект представляет собой интеллектуального RAG-агента (Retrieval-Augmented Generation) для технической поддержки операторов ЧПУ станков. Бот интегрирован во ВКонтакте и использует гибридный поиск (векторный по смыслу + регулярные выражения по кодам ошибок) по официальным сервисным руководствам Heidenhain.


---

## 🛠 Архитектура проекта
* **Оркестратор:** n8n (Self-Hosted)
* **LLM Модели:** Gemini 2.0 / Groq (Llama 3)
* **Векторная база данных:** Встроенная память n8n (Simple Vector Store с Memory Key)
* **Интерфейс (Frontend):** ВКонтакте Callback API
* **Туннелирование:** VPS + Nginx + SSL (или ngrok для локального развертывания)

---

## ⚙️ Инструкция по запуску (Быстрый старт)

Для развертывания проекта на вашем сервере или локальной машине выполните следующие шаги:

### Шаг 1: Импорт логики в n8n
В папке `workflows` данного репозитория лежат два JSON файла.
1. Запустите ваш экземпляр **n8n**.
2. В интерфейсе n8n нажмите `Add Workflow` -> кнопка `...` в правом верхнем углу -> `Import from File`.
3. Загрузите сначала вспомогательный воркфлоу **Exact Search Tool.json**, активируйте его (переключатель Active / Publish) и скопируйте его ID из адресной строки.
4. Загрузите основной воркфлоу **Main AI Agent.json**. В ноде `Call Workflow (Tool)` вставьте скопированный ID вспомогательного воркфлоу.

### Шаг 2: Настройка доступов (Credentials)
В основном воркфлоу n8n вам потребуется создать и заполнить доступы для нейросетей:
* **LLM API Key:** Добавьте рабочий ключ от Google Gemini или Groq (в зависимости от выбранной ноды).

### Шаг 3: Настройка интеграции ВКонтакте
1. Создайте группу ВК, перейдите в **Управление** -> **Работа с API**.
2. Создайте **Ключ доступа (Access Token)** с правами на управление сообществом и сообщения. Вставьте этот токен в ноду `HTTP Request` (параметр `access_token`) в n8n.
3. Перейдите во вкладку **Callback API**. 
4. Скопируйте Test URL из стартовой ноды `Webhook` в n8n. *(Важно: URL должен быть публичным HTTPS. Используйте ngrok для локальной разработки или настройте Nginx reverse-proxy на сервере).*
5. Вставьте URL в настройки ВК и подтвердите сервер. Для этого в n8n предусмотрена ветка `Switch` -> `Respond to Webhook`, которая автоматически вернет проверочный код.
6. В разделе "Типы событий" в ВК поставьте галочку на **Входящие сообщения**.

### Шаг 4: Инициализация базы знаний (Единоразово после запуска)
В проекте используется `Simple Vector Store` с функцией сохранения в оперативной памяти (Memory Key: `Heidenchain_clean_1`). 
Перед первым использованием бота необходимо загрузить мануал:
1. Откройте воркфлоу-загрузчик (прилагается в папке `workflows`).
2. Нажмите **Execute Workflow**.
3. Нода прочитает PDF файл мануала, разобьет его на чанки, сделает эмбеддинги и сохранит в памяти n8n для доступа Агента.
4. Система готова к работе!

## 🔍 Демонстрация работы

Бот поддерживает два режима поиска, которые выбираются LLM-агентом автоматически с помощью паттерна Chain-of-Thought:
* **Точный поиск по коду:** Пользователь пишет `Столкнулся с ошибкой 330-0042. Что делать?` -> Агент использует скрипт точного поиска по документу (через регулярные выражения), извлекает параметры и выдает пошаговый алгоритм решения.
* **Семантический векторный поиск:** Пользователь пишет `Как проверить модуль MC 6110?` -> Агент использует векторный поиск по смыслам, находит нужные абзацы в мануале и формирует технический ответ.

> **Примечание по безопасности:** Агент настроен на строгий запрет галлюцинаций. Если информации нет в сервисном мануале, бот не выдумывает решение, а честно рекомендует обратиться в официальную поддержку Heidenhain.
